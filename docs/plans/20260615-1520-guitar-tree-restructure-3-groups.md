# Plan: 樹狀圖結構重組 — 6 主幹合併為 3 大群

## 需求討論摘要
使用者認為原本 6 個主幹節點可以收斂成 3 組，語意上更清晰。
確認採用方案 A（技術入門 / 音樂知識 / 實踐應用）。

## 目標
將原本平行的 6 個主幹節點重組為 3 個語意群組，每群組各含 2 個原主幹作為子節點。

## 設計 Spec

### 新樹狀結構
```
🎸 電吉他學習
├─ 1. 技術入門
│   ├─ 基礎入門
│   │   ├─ 認識吉他結構
│   │   ├─ 持琴與姿勢
│   │   ├─ 調音
│   │   └─ 基礎樂理
│   └─ 基本技巧
│       ├─ 左手技巧
│       └─ 右手技巧
├─ 2. 音樂知識
│   ├─ 音樂理論
│   │   ├─ 音階
│   │   ├─ 和弦理論
│   │   └─ 節奏感訓練
│   └─ 音樂風格
│       ├─ Blues / Rock / Metal / Pop / Jazz
└─ 3. 實踐應用
    ├─ 設備知識
    │   ├─ 電吉他類型
    │   ├─ 音箱
    │   └─ 效果器
    └─ 實戰進階
        ├─ 每日練習計畫
        ├─ 即興演奏
        ├─ 錄音與創作
        └─ 合奏與舞台表演
```

### 節點命名
| 群組 | 名稱 |
|------|------|
| 群組 1 | `技術入門`（移除數字前綴） |
| 群組 2 | `音樂知識`（移除數字前綴） |
| 群組 3 | `實踐應用`（移除數字前綴） |

原主幹名稱（基礎入門、基本技巧…）降一層，不加編號。

## 技術決策
- 只修改 JS data 物件的巢狀結構
- 不改 D3 layout 邏輯，因為只是資料層的重組
- 原 6 個主幹 `name` 欄位去掉數字前綴（`1. 基礎入門` → `基礎入門`）

## 實作步驟
1. 重寫 `data.children` 為 3 個群組節點
2. 每個群組的 `children` 包含對應的 2 個原主幹（children 結構不動）
3. 原主幹名稱移除數字前綴

## 預期產出
- `guitar-learning-tree.html` data 物件更新，樹狀圖從 6 主幹改為 3 主幹各含 2 子幹
